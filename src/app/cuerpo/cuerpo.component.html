- ¿Qué es un framework?

Un framework es “un marco de trabajo que ofrece una estructura base para elaborar un proyecto con objetivos específicos”; en este caso, se podría decir que es “una especie de plantilla que sirve como punto de partida para la organización y desarrollo de software”.

Un framework permite “acometer un proyecto de software en menos tiempo”, consiguiendo tener un código más limpio y ordenado, consistente y reutilizable, ya que muchos elementos están estructurados como módulos que se puede reutilizar.

Un framework, en este caso Angular, tiene como aplicación u objetivo elaborar proyectos de programación, en este caso desarrollar aplicaciones de tipo web.

Angular es un framework de JavaScript de código abierto escrito en TypeScript. Esto quiere decir que el propio framework de Angular está programado en TypeScript, pero el usuario que emplea dicho framework programa en JavaScript.

Para trabajar con Angular, el usuario deberá estar familiarizado con los lenguajes HTML, CSS y JavaScript. Además, el usuario deberá tener instalado en su equipo el gestor de paquetes Node.js (npm) para ejecutar y desplegar las herramientas de JavaScript fuera del navegador web.

- ¿Cómo instalar Angular?

Para instalar Angular, tal y como hemos comentado en el apartado anterior, debemos instalar el gestor de paquetes Node.js (npm) y, a través de él, instalar el framework de Angular.

Lo primero de todo, para la instalación de Angular, deberemos acceder a la página web de Node.js e instalar dicho gestor de paquetes.

Una vez instalado Node.js, si tenemos Windows como O.S, deberemos abrir PoweShell para habilitar la ejecución de Scripts en el equipo y poder instalar de manera correcta Angular. En PoweShell deberemos lanzar el siguiente comando: 


            
            
                # Set-ExecutionPolicy -Scope CurrentUser -ExecutionPolicy RemoteSigned

Tras ello, deberemos abrir una consola de comandos (CMD o Terminal) y lanzar el siguiente comando en ella: 


            
            
                $ npm install -g @angular/cli

En este caso, tras la instalación, podemos ver que estamos trabajando con las últimas versiones de:
- Angular (v19.0.6)
- Node.js (v22.12.0)
- npm (v11.0.0)
- Windows (v11 x64 bit) 


            
            
                $ ng v

Tras la instalación, trabajaremos con las siguientes variables de entorno para Node y Angular: 


            
            
                C:\Program Files\nodejs
                C:\Users\your_username\AppData\Roaming\npm\node_modules\npm\bin
                C:\Users\your_username\AppData\Roaming\npm\node_modules\@angular\cli\bin

- Creando nuestra primera aplicación Angular (Hello, World!)

Para crear una aplicación base en Angular, una vez instalado el framework, lo primero que deberemos hacer es abrir la consola de comandos (CMD o Terminal) y lanzar el siguiente comando en ella: 


            
            
                $ ng new --no-standalone my-app

Una vez creada la aplicación, nos situaremos en el workspace de dicha app mediante el siguiente comando: 


            
            
                $ cd new my-app

Una vez hechos todos los pasos seguidos anteriormente, tan solo debemos ejecutar el siguiente comando desde la consola de comandos (CMD o Terminal) para poder visualizar la aplicación desde el navegador: 


            
            
                $ ng serve --open

También exporta el título o nombre de la app (my-app) en una variable hacia el fichero app.component.html mediante el export final.

Otro fichero importante es app.modules.ts. Este fichero se genera al hacer una instalación atendida (parámetro --no-standalone) que hemos utilizado antes al generar la aplicación. Es interesante utilizar este parámetro porque, por ejemplo, todos los componentes que generemos nuevos en nuestra aplicación se declararán automáticamente en este fichero. - Creando componentes en Angular...

Una vez que hemos hecho nuestra primera aproximación a Angular creando nuestra primera aplicación (Hello, World!), vamos a crear una aplicación más avanzada. Para ello, vamos a crear un nuevo componente en Angular ejecutando el siguiente comando desde la consola de comandos (CMD o Terminal): 


            
            
                $ ng generate component my-component

Como podemos ver en la foto anterior, se ha generado un nuevo directorio llamado my-component. En los ficheros HTML y CSS de este directorio será donde trabajaremos con el código HTML y CSS. De la misma manera, se ha actualizado el fichero app.module.ts. En este fichero se ha referenciado de manera automática el nuevo componente creado.

Una vez hemos creado nuestro nuevo componente, procedemos a editar los ficheros HTML y CSS del nuevo componente my-component creado. Los vamos mostrando uno a uno.

Lo primero de todo, mostramos el fichero my-component.component.ts del nuevo componente creado. Este fichero no hace falta editarlo, pues contiene referenciados los ficheros HTML y CSS, en los cuales se editará el código HTML y CSS para este componente.

Tras este fichero, tenemos el fichero my-component.component.html, en el cual se editará el código HTML.

Por último, tenemos el fichero my-component.component.css, en el cual se editará el código CSS.

Una vez tenemos editados todos estos ficheros, tal y como podemos ver en las capturas anteriores, tenemos que volver al fichero HTML del componente component por defecto (app.component.html) para refrenciar este nuevo componente creado. Esta es la idea principal de Angular, el del trabajo por componentes, lo cual hace que ganemos en modularidad. Referenciarlo será tan sencillo como implementar el siguiente tag: 


            
            
                <app-my-component></app-my-component>

Tras hacer lo comentado anteriormente, ya estamos listos para desplegar la aplicación y visualizarla desde nuestro navegador web: 


            
            
                ng serve --open
